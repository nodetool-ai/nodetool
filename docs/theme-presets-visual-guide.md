## Color Usage Guide

### Text Colors
All themes include three text color levels:
- **Primary**: Main text content
- **Secondary**: Less prominent text
- **Disabled**: Inactive elements

### Status Colors
Consistent across all themes:
- **Error**: Red tones for errors and destructive actions
- **Warning**: Orange/yellow tones for warnings
- **Info**: Blue tones for information
- **Success**: Green tones for success states

### Action States
Each theme defines colors for:
- **Hover**: Subtle highlight on interactive elements
- **Selected**: Background for selected items
- **Focus**: Focus ring for keyboard navigation
- **Disabled**: Appearance of disabled elements

### NodeTool-Specific Colors
Custom colors for NodeTool elements:
- **Node backgrounds**: Different for regular nodes and group nodes
- **Editor grid**: Canvas grid and axis colors
- **Providers**: API, Local, and HuggingFace badge colors
- **Connections**: Input and output connection colors

## Choosing the Right Theme

### For Long Coding Sessions
- **Ocean** - Cool tones reduce eye strain
- **Forest** - Natural colors are calming

### For Creative Work
- **Sunset** - Warm, energetic colors inspire creativity
- **Midnight** - Sophisticated tones for premium feel

### For Accessibility
- **Light** - High contrast for bright environments
- **Dark** - Classic dark mode for low-light settings

### For Personal Preference
Try each theme and see which colors resonate with you! Your comfort and enjoyment are most important.

---

## Implementation Details

Each theme uses the Tailwind CSS color palette as a foundation, ensuring:
- Consistent color relationships
- Proven accessibility
- Beautiful color harmonies
- Professional appearance

All themes maintain the same semantic meaning for colors (e.g., red always means error) while varying the specific hues to match the theme's character.
