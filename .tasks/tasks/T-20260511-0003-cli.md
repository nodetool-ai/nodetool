# Description

Provide `npm run task` as the standard interface for creating and
moving tasks. The CLI is the only thing that should mint task IDs
(to keep filenames unique) and the only thing that should bump
`updated:` timestamps.

# Acceptance criteria

- [x] `task list [--state=X] [--plan=X] [--json]`
- [x] `task new plan --title=...`
- [x] `task new task --plan=... --title=...` auto-assigns next free counter
- [x] `task transition <id> <state>` enforces allowed transitions
- [x] `task validate` checks frontmatter, references, sections, checkboxes
- [x] Zero npm dependencies (hand-rolled YAML parser for our schema subset)
