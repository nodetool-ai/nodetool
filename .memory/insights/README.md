# Insights and Best Practices

This folder contains architectural insights, best practices, and important learnings organized by topic. Each file documents significant discoveries that help agents write better code.

## Folder Structure

```
insights/
├── architecture/     - Architectural patterns and decisions
├── performance/      - Performance optimization techniques
├── testing/          - Testing strategies and patterns
├── code-quality/     - Code quality best practices
├── ui-ux/           - UI/UX patterns and guidelines
├── api-backend/     - API and backend integration patterns
├── build-system/    - Build and development tooling
├── deployment/      - Deployment and distribution
└── future/          - Future considerations and upgrade paths
```

## File Naming Convention

Use descriptive, kebab-case names that summarize the insight:
- `zustand-selective-subscriptions.md`
- `virtual-scrolling-for-large-lists.md`
- `testing-library-patterns.md`

## File Format

Each insight file should follow this format:

```markdown
# Insight Title

**Insight**: What was learned or discovered

**Rationale**: Why it matters

**Example**: Code example (if applicable)

**Impact**: Measurable benefit (if known)

**Files**: Related files (optional)

**Date**: YYYY-MM-DD (when documented)
```

## Adding New Insights

1. Identify the correct topic folder
2. Create a file with a descriptive name
3. Follow the format above
4. Focus on actionable, reusable knowledge
