# NodeTool v0.6.3-rc.16 Changelog

(including changes from rc.12 - rc.16)

## New Features

### Image Editor [beta]

- **Integrated Image Editor**: Edit images directly within the asset panel with full backend persistence.
- **Drawing & Annotation Tools**: Added Fill, Text, Rectangle, Ellipse, Line, and Arrow tools for basic image editing.

### Workflow Execution

- **Job Duration Display**: Workflow completion notifications now show how long the job took to run.
- **Live Running Time**: The FloatingToolBar now displays elapsed time during workflow execution.
- **Checkpoint Manager**: Zero-overhead checkpoint system enables resumable workflows.

### Developer Tools

- **CLI Installer**: New shell-based installer with micromamba bootstrap for easier command-line setup.

## Improvements

### Node Connections

- **Smarter Type Matching**: Node connection suggestions are now sorted by match quality for both input and output types, making it easier to find compatible connections.

### HuggingFace Integration

- **Improved Flux Cancellation**: Enhanced cancellation handling for Flux inference using native interrupt flags for more responsive workflow stops.

### Configuration

- **HuggingFace Cache Directory**: Environment configuration now supports custom HuggingFace cache paths.
- **Conda Environment Persistence**: Default Conda environment settings now persist correctly.
- **Python 3.11 Requirement**: Minimum Python version updated to 3.11.

## Bug Fixes

- Fixed macOS permission denied error when spawning uv from Electron.
- Fixed transform tools in the image editor.
- Fixed JavaScript runner syntax issues.
- Improved error handling for Python environment checks.
- Force-release GPU lock during job cancellation to prevent deadlocks.
- Enhanced logging for GPU lock acquisition timeouts.
- Prevented redundant job completion updates.

## Documentation

- Added Image Editor documentation.
- Added VibeCoding feature documentation.
- Added mobile app documentation page.
- Updated quickstart to recommend Qwen Image instead of SDXL.

## Testing

- Added 200+ new tests covering e2e workflows, hooks, stores, and node modules.
- Fixed flaky e2e tests in Electron and Web packages.
